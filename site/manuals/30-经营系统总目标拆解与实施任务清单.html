小黑托昵经营系统总目标拆解与实施任务清单¶

版本：v1.0
更新日期：2026-03-14
当前原则：先做全量真实数据 -> 再做经营判断 -> 再做使用体验
当前不优先处理：失败报表用上次成功数据兜底

1. 当前总目标¶

这个项目不是单纯做一个报表页面，而是要逐步做成一个门店能每天使用的经营系统。

目标拆成 5 句话：

  1. 老板娘 打开首页就知道今天先点哪里、先做什么
  2. 店长 能用详细页、月度页、季度页开经营会和盯执行
  3. 店员 能拿着系统里的建议去卖货、补货、去化、做连带
  4. 店里电脑 能通过一键同步抓取 Yeusoft 全量报表并刷新看板
  5. 线上 GitHub Pages 永远保持可看、清楚、稳定，不依赖线上抓取

2. 三种任务拆分方式¶

方案 A：按页面拆¶

适合： - 先把首页、仪表盘、详细页、月度页、季度页一个个做完整

优点： - 好理解 - 页面体验提升很快

缺点： - 容易只优化展示，不解决底层数据真实性 - 一键同步、接口、配置这些底层工作容易被拖后

方案 B：按能力链路拆¶

适合： - 先做数据，再做分析，再做页面，再做本地控制台

优点： - 技术逻辑清楚 - 后面扩展最稳

缺点： - 阶段成果对非技术人员不够直观 - 前几步“看起来没什么变化”

方案 C：按门店使用链路拆（最推荐）¶

适合： - 从老板娘每天真的怎么用出发推进

链路是： 1. 先同步 2. 再看结论 3. 再看详细分析 4. 再拍板补货/去化/陈列/会员动作 5. 再复盘月度和季度

优点： - 最贴近日常使用 - 数据、页面、建议、按钮会自然协同 - 不会做出“功能很多但不好用”的系统

缺点： - 需要同时管页面和数据 - 任务拆解要更细

3. 最推荐的任务主线¶

后续按 6 条主线推进：

主线 1：全量同步引擎¶

目标： - 确保数据尽量真实、完整、可验证

任务： 1. 列出所有 Yeusoft 报表 2. 给每张报表标记： - 是否已打通 - 是否支持全量区间 - 是否分页 - 是否值得引入看板 3. 优先打通 A 类高价值报表全量同步 4. 让本地一键同步跑完后产出清晰状态 5. 记录： - 成功 - 部分成功 - 失败 - 时间范围 - 行数

完成标准： - 能清楚知道“这次到底同步了哪些报表，哪些没同步”

主线 2：老板首页与仪表盘¶

目标： - 老板娘 10 秒内知道今天该干什么

任务： 1. 首页继续保持最短路径： - 进仪表盘 - 进详细页 - 本地同步 2. 仪表盘首页继续只放最重要模块 3. 增加更明确的： - 今日结论 - 上次同步时间 - 当前经营阶段 - 本月经营状态 - 待处理事项 4. 所有提示文案继续去技术化

完成标准： - 老板娘不用判断页面结构，也知道先点什么

主线 3：详细经营分析¶

目标： - 让系统不只是“展示数据”，而是像零售顾问一样给动作

任务： 1. 继续完善： - 销售表现分析 - 品类结构分析 - SKU 分析 - 库存健康分析 - 会员与客单分析 - 节奏与目标分析 2. 按角色拆建议： - 老板娘 - 店长 - 店员 3. 建立“动作优先级排序” 4. 让长解释进入 tips 或折叠区，保持页面可读

完成标准： - 店长可以直接拿页面内容开晨会或周会

主线 4：月度页与季度页¶

目标： - 从“今天怎么干”延伸到“这个月和这个季度怎么经营”

任务： 1. 持续增强月度页： - 月销售趋势 - 月毛利 / 净利 / 保本 - 月库存结构 - 月进货建议 2. 持续增强季度页： - 季度结构变化 - 季度品类贡献 - 季度去库存与换季建议 3. 月度 / 季度建议继续跟真实数据联动，不写空话

完成标准： - 老板能直接判断“这个月是冲销售、稳利润、去库存，还是调结构”

主线 5：成本、利润与保本¶

目标： - 看板里的利润预测尽量接近真实经营

任务： 1. 固定费用、工资、毛利率继续形成稳定口径 2. 毛利率优先用真实数据，缺失时再回退默认值 3. 保本销售额、保本进度、月末净利预测继续优化 4. 成本维护台继续做成非技术人员可维护

完成标准： - 老板看到利润和保本时，不会因为一堆 0 值失真

主线 6：本地控制台与操作稳定性¶

目标： - 让“本地一键抓取并刷新”真正成为店里电脑可依赖的操作入口

任务： 1. 同步状态更清楚： - 是否连接本地服务 - 是否正在抓取 - 哪些报表成功 - 哪些失败 2. 首页和成本维护台入口继续优化 3. 操作失败时说人话 4. 后面再考虑失败报表兜底

完成标准： - 老板娘知道“现在能不能同步、同步到哪一步了、接下来该怎么办”

4. 当前优先级排序¶

重要且紧急¶

  1. 继续提升 全量同步 的真实度
  2. 继续接入高价值报表到经营分析
  3. 让月度 / 季度页建议和真实数据更紧密

重要不紧急¶

  1. 优化本地控制台状态
  2. 继续优化利润口径
  3. 做更稳的配置和本地服务地址管理

可观察¶

  1. 首页文案进一步极简
  2. 图表样式继续微调
  3. 文档中心继续整理

暂不处理¶

  1. 失败报表用上次成功数据兜底
  2. 更重的前端框架重构
  3. 远程云端抓取

5. 下一阶段建议顺序¶

建议按这个顺序往下做：

  1. 全量同步引擎继续补强
  2. 更多高价值报表接进经营分析
  3. 月度页 / 季度页更像经营复盘页
  4. 本地控制台状态更清楚
  5. 成本和利润口径继续逼近真实

6. 这份清单怎么用¶

如果后面继续推进，建议每次都先判断我们在做哪条主线：

  • 如果是在补报表和接口：归到 主线 1
  • 如果是在补首页和仪表盘：归到 主线 2
  • 如果是在补建议和分析：归到 主线 3
  • 如果是在补月度季度：归到 主线 4
  • 如果是在补利润成本：归到 主线 5
  • 如果是在补本地抓取和同步体验：归到 主线 6

这样仓库越做越大时，仍然不会乱。
