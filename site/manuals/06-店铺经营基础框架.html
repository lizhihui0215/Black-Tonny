小黑托昵店铺经营基础框架¶

版本：v1.2
更新日期：2026-03-13
适用对象：单城市门店、线下成交为主、抖音内容作为引流补充
仓库定位：这份文档把项目从“只做抖音号”扩展为“内容增长 + 店铺经营”的双主线文档仓

使用说明¶

  • 这份文档解决的是 店铺经营怎么稳，不是单纯解决抖音怎么发。
  • 默认当前门店的增长结构分成两条线：
  • 内容增长线：抖音起号、老号重整、AI 学习、内容执行
  • 店铺经营线：进店接待、产品结构、复购、会员、库存、日常经营
  • 先把抖音流量导进店，再把店内接待和复购做稳，才算项目真正跑起来。

1. 店铺经营的总目标¶

1.1 核心目标¶

店铺经营不只看当天卖了多少，而是看这 4 件事是否同时在变好：

  1. 进店成交是否更稳
  2. 客单是否更清楚
  3. 老客复购是否更明显
  4. 门店对抖音流量的承接是否更顺

1.2 这个仓库未来的双主线¶

后面整个仓库都按这两个系统推进：

A. 内容增长系统¶

  • 负责获客
  • 负责建立“老板娘可信”的认知
  • 负责让宝妈主动问地址、尺码、来店时间

B. 店铺经营系统¶

  • 负责成交
  • 负责服务体验
  • 负责复购
  • 负责把流量变成可持续经营

结论：

  • 抖音不是目的，是门店经营的一部分
  • 门店经营也不能脱离内容，因为内容是新的获客入口

2. 店铺经营的 6 个核心模块¶

2.1 进店接待¶

目标：

  • 顾客一进店，尽快知道她是随便看看，还是带着明确问题来的
  • 如果是抖音来的，要立刻接上她在线上提过的问题

最小 SOP：

  1. 先问：是自己看看，还是我先帮你缩一下厚薄和尺码？
  2. 如果对方提到抖音： - 接话：那我先按你线上说的情况，带你从这两类开始看
  3. 不先讲品牌故事，先讲怎么选

2.2 产品结构¶

目标：

  • 不让货很多但顾客还是不会选
  • 让老板娘能快速把顾客引导到正确范围

建议先按用户决策去整理，不先按品牌话术去整理：

  1. 换季厚薄
  2. 年龄段
  3. 尺码
  4. 居家 / 通勤 / 活动量

2.2A 当前最适合门店的产品矩阵¶

基于公开可见商品线，门店经营和内容表达统一按 儿童贴身棉品 来理解，优先覆盖这 5 类：

  1. 内裤
  2. 背心 / 发育期近身打底
  3. 秋衣秋裤 / 单裤 / 打底
  4. 家居服 / 睡衣
  5. 袜品 / 连裤袜

门店不要把这 5 类平均对待，优先级建议是：

  1. 秋衣秋裤 / 打底 最容易接住换季、厚薄、尺码问题
  2. 内裤 最容易接住舒适感、尺码、复购问题
  3. 家居服 最容易接住场景和体感问题
  4. 背心 最容易接住年龄段和过渡期问题
  5. 袜品 更适合作为补充成交和内容搭配，不建议单独做主线

2.2B 产品结构的门店决策树¶

顾客进店后，优先不是问“你想买哪一款”，而是先分到哪一类：

  1. 换季问题 先看秋衣秋裤、打底、家居服
  2. 舒适和贴身感问题 先看内裤、背心
  3. 尺码和试穿问题 先看秋衣秋裤、内裤、背心
  4. 补充搭配问题 再带到袜品

判断是否做得好：

  • 顾客进店后 3-5 分钟内，老板娘能帮她把范围明显缩小

2.3 成交承接¶

目标：

  • 不让顾客“看懂了视频”，却“进店后还是不知道怎么买”

门店承接重点只保留 3 个：

  1. 先判断厚薄
  2. 再缩尺码
  3. 最后再选具体款

这和抖音内容必须一致。
如果线上讲的是“怎么选”，线下却一上来推活动，转化会掉。

2.4 老客复购¶

目标：

  • 不是等顾客自然回头，而是主动识别哪些顾客值得做复购经营

最值得记录的老客类型：

  1. 已经在店里买过贴身棉品
  2. 对老板娘建议认可度高
  3. 购买后很少退换纠结
  4. 会再带孩子或再次来店

经营动作：

  • 把老客高频问题沉淀回内容
  • 把内容里的高频问题反过来用于老客接待

2.5 会员与私域沉淀¶

目标：

  • 不让抖音流量和门店顾客只停留在一次成交

建议门店记录 3 类顾客：

  1. 抖音来的
  2. 老客复购
  3. 自然进店

哪怕前期不做复杂系统，也至少要做到：

  • 知道今天有多少抖音来的顾客
  • 知道哪些是值得二次触达的顾客

2.6 日常经营基本盘¶

每天必须看的不只是营业额，还包括：

  1. 到店人数
  2. 成交人数
  3. 抖音到店人数
  4. 问尺码 / 地址 / 到店时间的人数
  5. 老客复购人数
  6. 哪类产品最近最常被问

3. 抖音和店铺经营怎么真正打通¶

3.1 内容负责把问题带进来¶

抖音内容最理想的结果，不是播放高，而是把这 3 类问题带进店：

  1. 店在哪
  2. 孩子穿什么码
  3. 最近适不适合来

结合当前产品线，优先把下面这 3 类问题带进店：

  1. 秋衣秋裤 / 打底现在怎么选厚薄
  2. 内裤 / 背心到底怎么缩尺码
  3. 家居服 / 袜品适不适合顺手一起看

3.2 门店负责把问题接住¶

如果顾客进店后，店员或老板娘接不上这 3 类问题，内容再好也会浪费。

所以线下必须做到：

  • 看过抖音来的顾客，先接线上问题
  • 接待时沿用线上同样的话术逻辑
  • 把线下高频问题再送回抖音做内容

3.3 最小闭环¶

固定闭环就是：

  1. 抖音内容提出真实问题
  2. 顾客评论 / 私信 / 到店
  3. 门店按同样逻辑完成接待
  4. 老客或到店反馈再回到内容里

4. 店铺经营未来 90 天怎么推进¶

Phase A（Day 1-30）：先把接待和记录做稳¶

目标：

  • 让门店能接住抖音来的流量
  • 把日常记录从“凭感觉”变成“有简单数据”

必须完成：

  1. 到店接待统一话术
  2. 每天记录抖音到店、地址咨询、尺码咨询
  3. 先分清哪类产品和哪类顾客最值得复购经营

Phase B（Day 31-60）：开始做复购和产品结构优化¶

目标：

  • 让老客经营开始形成节奏
  • 让货架和讲解方式更贴近顾客决策，而不是只贴近品牌陈列

必须完成：

  1. 老客复购记录表
  2. 产品整理逻辑按“怎么选”而不是“怎么摆”
  3. 把最常被问的 5 类产品问题写成接待话术

Phase C（Day 61-90）：做内容与门店的一体化经营¶

目标：

  • 把抖音和店铺变成同一个经营系统，而不是两个分开的动作

必须完成：

  1. 能明确说出：哪类内容最能带来哪类到店顾客
  2. 能明确说出：哪类到店问题最值得再拍成内容
  3. 把门店接待、复购、内容题型形成固定循环

5. 每天 / 每周 / 每月建议动作¶

每天¶

  • 记录抖音到店人数
  • 记录强购买问题数量
  • 记录老客复购人数
  • 记录最常被问的 1 个问题

每周¶

  • 复盘哪类内容带来咨询
  • 复盘哪类顾客最常成交
  • 更新一版评论 / 私信 / 接待话术

每月¶

  • 复盘产品结构是否需要调整
  • 复盘哪些顾客值得重点做复购经营
  • 复盘抖音和店铺之间是否真的形成闭环

7. 当前结论¶

这个仓库后面不应该再被理解成“抖音号项目”。
更准确的定位应该是：

小黑托昵门店增长与经营文档仓

其中：

  • 抖音负责获客和建立信任
  • 门店负责接待、成交和复购
  • 两条线必须持续互相喂数据、喂问题、喂内容
