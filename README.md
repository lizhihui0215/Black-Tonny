## 仓库定位

后续这个仓库统一按三条主线维护：

- `内容增长线`：负责获客、建立信任、把问题带进门店
- `店铺经营线`：负责接待、成交、复购、经营稳定
- `小团队管理线`：负责 `1个老板 + 2个员工` 的分工、排班、带教、复盘和执行稳定

三条线必须互相联动，而不是各做各的。

## 维护规则

- 改策略、SOP、脚本、学习路径、经营动作时，优先改 `docs/`
- 改了章节结构后，同步更新 `mindmaps/`
- 新增专题时，沿用两位序号命名，方便长期排序
- 如果 XMind 文件需要继续补，优先保持和同主题 Markdown 脑图一致
