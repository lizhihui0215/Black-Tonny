# POS 报表收益评估与接入建议

## 这份文档解决什么问题

Yeusoft 的报表不少，但不是每张都值得进老板仪表盘。

这份文档的目标是先把报表按经营价值分级：

- `A 类`：直接提升老板每天决策，值得优先接入看板
- `B 类`：适合详细页、周复盘、专项分析，不一定放首页
- `C 类`：暂时只做参考，不优先投入抓取和接入

## 当前状态

- 已有成功样本：
  - `店铺零售清单`
  - `销售清单`
- 已确认 `GetMenuList` 可以稳定返回报表的 `FuncLID / FuncUrl / FuncName`，后续不必继续强依赖左侧浮层菜单点击。
- 已验证可以直接打开并抓到真实数据接口的高价值报表：
  - `库存综合分析`
  - `出入库单据`
  - `每日流水单`
- 当前自动抓取的主要阻塞，已经从“找不到菜单”缩小成“不同报表对应不同 API 家族，需要分组接入”。

## 已验证可直接调数的高价值报表

### 1. 库存综合分析

已确认链路：

- 菜单入口：`FuncLID = E004002003`
- 打开方式：`FuncUrl = StockAnalysis`
- 直连数据接口：`POST /eposapi/YisEposReport/SelStockAnalysisList`

价值：

- 直接支撑 `库存压力`、`季节库存结构`、`去化优先级`
- 很适合做首页的 `库存风险` 和 `补货机会`

### 2. 出入库单据

已确认链路：

- 菜单入口：`FuncLID = E004003002`
- 打开方式：`FuncUrl = InOutDoc`
- 直连数据接口：`POST /eposapi/YisEposReport/SelOutInStockReport`

价值：

- 很适合做 `负库存纠偏`、`异常调拨`、`入库/退货追踪`
- 能直接提升库存准确性判断

### 3. 每日流水单

已确认链路：

- 菜单入口：`FuncLID = E004006001`
- 打开方式：`FuncUrl = dailyFlow`
- 直连数据接口：`POST /JyApi/ReconciliationAnalysis/SelectRetailDocPaymentSlip`

价值：

- 能直接补 `支付结构`、`流水节奏`、`利润与保本`
- 是老板看板升级为经营驾驶舱的关键来源

补充说明：

- 这个页面在前端组件上有一个已出现的 created hook 报错，所以页面 tab 状态不完全稳定。
- 但数据接口已经真实发出，说明这张表仍然值得优先按 API 接入，而不是继续依赖前端页面状态。

## A 类：建议优先接入老板仪表盘

### 1. 店铺零售清单

经营价值：

- 能直接看主店零售明细
- 能按 `输入人 / 店铺名称` 区分主店和参考店
- 适合判断 `本店真实成交`、`其他店铺参考`、`导购承接`

建议接入：

- 首页 `赚钱机会`
- 首页 `会员经营`
- 详细页 `会员 / 店员 / 参考店`

### 2. 销售清单

经营价值：

- 更接近真实销售流水
- 适合看 `日销趋势`、`订单变化`、`活动前后波动`
- 适合补足首页的 `今日经营重点`

建议接入：

- 首页 `今日经营重点`
- 首页 `核心经营指标`
- 详细页 `总览`

### 3. 库存综合分析

经营价值：

- 这是库存线里最贴近老板决策的一张
- 能直接支持 `库存压力`、`去化优先级`、`补货优先级`
- 是库存看板里的核心底表之一

建议接入：

- 首页 `库存风险`
- 首页 `补货机会`
- 详细页 `补货 / 去化`

### 4. 出入库单据

经营价值：

- 对负库存纠偏、盘点异常、调拨异常很关键
- 适合判断“库存不准是卖掉了，还是单据没过、调拨没记、退货没冲”

建议接入：

- 详细页 `补货 / 去化`
- 异常提醒：`负库存先纠偏`

### 5. 商品销售情况

经营价值：

- 适合拉长周期看历史累计
- 适合判断哪些品类长期值得补、哪些只是短期波动
- 对 `季节切换` 和 `产品结构调整` 很有价值

建议接入：

- 详细页 `总览`
- 详细页 `经营策略`

### 6. 每日流水单

经营价值：

- 对利润、现金回笼、支付结构、日经营节奏最直接
- 是把老板仪表盘从“销售库存看板”升级成“经营驾驶舱”的关键数据

建议接入：

- 首页 `利润与保本`
- 首页 `今日经营重点`
- 详细页 `总览`

### 7. 会员综合分析

经营价值：

- 适合判断会员活跃度、复购结构、沉睡会员
- 比单纯看消费排行更适合做会员运营

建议接入：

- 首页 `会员经营`
- 详细页 `会员 / 店员`

### 8. 会员消费排行

经营价值：

- 适合快速抓高价值会员
- 对老板和店员来说很容易理解，也容易执行

建议接入：

- 首页 `会员经营`
- 今日任务：`联系高价值会员`

## B 类：适合详细页或复盘，不一定放首页

### 零售明细统计

适合：

- 看更细的零售结构
- 做专项问题排查

不建议直接放首页原因：

- 颗粒度太细，老板第一屏不需要

### 库存明细统计

适合：

- 做单款、单色、单尺码级别复盘

不建议直接放首页原因：

- 更适合执行层下钻，不适合老板第一屏

### 库存零售统计

适合：

- 对比库存和零售联动关系
- 辅助判断品类节奏

不建议直接放首页原因：

- 适合做分析，不如 `库存综合分析` 直接

### 进销存统计

适合：

- 看阶段性进货、销售、库存的整体平衡

不建议直接放首页原因：

- 更适合月度复盘或进货决策，不是每天先看

### 商品品类分析

适合：

- 看品类结构是否偏了
- 判断扩品和压缩品类方向

不建议直接放首页原因：

- 更适合周 / 月经营策略

### 门店销售月报

适合：

- 看月度趋势
- 看淡旺季和节点波动

不建议直接放首页原因：

- 更新频率天然偏低

### 导购员报表

适合：

- 看导购表现
- 做排班、培训、带教和分工优化

不建议直接放首页原因：

- 更适合详细页和周复盘

## C 类：暂时参考，不优先接入老板看板

### 储值按店汇总

原因：

- 如果当前门店储值运营不是主线，价值有限

### 储值卡汇总

原因：

- 偏财务和储值体系管理，不是首页决策重点

### 储值卡明细

原因：

- 太细，更适合专项核对

## 最推荐的接入顺序

### 第一批

1. `库存综合分析`
2. `出入库单据`
3. `每日流水单`

原因：

- 这 3 张表最能直接提升老板每天的经营判断
- 一张解决库存压力
- 一张解决库存准确性
- 一张解决利润与现金节奏
- 现在也已经是最接近“直接调 API 接入看板”的 3 张

### 第二批

1. `会员综合分析`
2. `会员消费排行`
3. `导购员报表`

原因：

- 这组更偏执行和复购
- 对营业额增长很重要，但优先级低于库存和利润

### 第三批

1. `商品品类分析`
2. `门店销售月报`
3. `进销存统计`

原因：

- 这组更偏策略层
- 更适合周 / 月复盘，不急着放老板首页

## 当前接入建议

在 Yeusoft 抓取还没完全稳定之前，最值得优先攻克的是：

1. `库存综合分析`
2. `出入库单据`
3. `每日流水单`

只要这 3 张表抓稳，老板仪表盘就会从：

- `库存销售看板`

升级成：

- `库存 + 纠偏 + 利润` 的老板经营仪表盘

这会比继续往首页塞更多零散报表更值。
