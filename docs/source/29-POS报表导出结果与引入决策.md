# POS 报表导出结果与引入决策

## 这份文档解决什么问题

这份文档不是再讲“理论上哪些报表有价值”，而是把当前已经实际跑过的导出结果收成一份可决策清单：

- 哪些报表已经能导出
- 哪些报表虽然有价值，但还没有稳定拿到
- 哪些报表暂时收益不高，不值得优先投入

目标是让老板只回答一件事：

`哪些现在就接入看板，哪些先放一放。`

## 当前结论先看这里

### 建议立即引入

1. `店铺零售清单`
2. `销售清单`
3. `商品销售情况`
4. `库存综合分析`
5. `出入库单据`
6. `会员消费排行`

原因：

- 这一批已经通过你提供的 `curl` 直连接口验证
- 都已经确认了请求口径，能明确传 `2025-03-01` 到 `2026-03-14`
- 其中 `店铺零售清单` 和 `销售清单` 已经拿到明显的全量结果，适合直接做主数据源
- 其余几张虽然部分返回结构还需要继续拆字段，但已经值得接入详细页或辅助决策

### 建议优先继续攻克

1. `日进销存`
2. `会员综合分析`
3. `门店销售月报`
4. `每日流水单`

原因：

- 不是拿不到接口，而是还有 `超时 / 返回空数据 / 返回结构待拆` 这类问题
- 一旦这几张稳定，老板看板会更完整，尤其是 `趋势 / 月度 / 当日流水`
- 但现在直接强接入，会让看板对“真实经营结果”的表达变得混乱

### 建议暂缓引入

1. `零售明细统计`
2. `导购员报表`
3. `库存明细统计`
4. `库存零售统计`
5. `库存多维分析`
6. `进销存统计`
7. `储值按店汇总 / 储值卡汇总 / 储值卡明细`

原因：

- 这批大多已经能抓到，但对老板首页的直接收益不高
- 更适合详细页下钻、复盘分析或后续专项页
- 现在先接进去，反而会让首页更吵

## 按老板价值分级

## A 类：值得直接进老板看板

### 1. 店铺零售清单

当前导出结果：

- 成功
- 明确带了 `BeginDate=2025-03-01`
- 明确带了 `EndDate=2026-03-14`
- 当前返回行数 `83877`

为什么值得引入：

- 它是现在最接近“主店真实零售全量”的一张
- 可以区分 `郭文攀主店` 和其他输入人参考店
- 能直接支撑销售、会员、店员、主店经营判断

建议接入位置：

- 首页：`核心经营指标`
- 首页：`会员经营`
- 详细页：`总览`
- 详细页：`会员 / 店员`

### 2. 库存综合分析

当前导出结果：

- 已经确认菜单和直连接口
- 已经抓到真实样本
- 但它更像“当前库存结构快照”，不是完整历史明细

为什么值得引入：

- 能直接告诉老板库存主要压在哪些年季、季节
- 适合判断 `先补什么`、`先停补什么`、`先去什么`

使用边界：

- 可以进老板看板
- 但不要把它误解成 `2025-03 到现在的完整历史销售趋势`

建议接入位置：

- 首页：`库存风险`
- 首页：`补货机会`
- 详细页：`补货 / 去化`

### 3. 每日流水单

当前导出结果：

- 已经确认菜单和直连接口
- 已经抓到真实 API 样本
- 当前更适合 `当天流水 / 当天支付结构 / 当天件单数`

为什么值得引入：

- 最适合老板判断“今天营业节奏好不好”
- 能帮助利润与保本更像经营驾驶舱，而不是月末记账

使用边界：

- 适合做当天经营
- 不建议当前直接拿它做全年趋势结论

建议接入位置：

- 首页：`今日经营重点`
- 首页：`利润与保本`
- 详细页：`总览`

## B 类：有价值，但要先把抓取链路打稳

### 1. 销售清单

当前情况：

- 菜单、Grid 结构已识别
- 当前批量扫描仍然超时

为什么它重要：

- 它最适合看日销趋势、订单变化、活动前后波动
- 如果能稳定抓全量，对首页结论帮助会非常大

为什么现在不直接重接入：

- 当前抓取不稳
- 强行接入会让首页判断受不完整数据影响

建议：

- 放在下一批优先攻克名单第一位

### 2. 商品销售情况

当前情况：

- 菜单已确认
- 当前批量扫描超时

为什么它重要：

- 它更适合看长周期累计销售
- 对判断“哪些品类长期值得补、哪些只是短期起量”很关键

建议：

- 适合作为第二批重点接入
- 稳定后优先放到 `详细页总览` 和 `经营策略`

### 3. 出入库单据

当前情况：

- 菜单和直连接口已确认
- 有真实样本
- 但当前样本更像短窗口，不足以确认 `2025-03` 到现在的全量已稳定导出

为什么它重要：

- 它直接决定库存准不准
- 对负库存、调拨、补货、退货异常判断非常重要

建议：

- 继续优先攻克
- 稳定后放到 `详细页补货 / 去化`
- 在完全稳定前，只把它作为辅助判断，不作为首页主结论来源

### 4. 会员综合分析

当前情况：

- 菜单已确认
- 还没有稳定样本

为什么它有价值：

- 它比单纯会员消费排行更适合判断活跃度、复购和沉睡会员

建议：

- 放到第二梯队
- 稳定后优先接到 `会员经营`

### 5. 会员消费排行

当前情况：

- 菜单已知
- 还没拿到稳定样本

为什么它有价值：

- 对老板很直观
- 很适合今天就去联系高价值会员

建议：

- 作为 `会员经营` 的补充，不必比 `会员综合分析` 更早

## C 类：当前收益不高，先不优先投入

### 零售明细统计

问题：

- 当前能打开页面，但没拿到真实结果集
- 就算拿到，也更适合排查细项，不适合老板首页

建议：

- 暂不优先

### 库存明细统计

问题：

- 当前能打开页面，但没拿到真实结果集
- 更像执行层下钻，不是老板第一屏需要的信息

建议：

- 暂不优先

### 库存零售统计

问题：

- 对分析有帮助
- 但在老板视角里，不如 `库存综合分析` 直接

建议：

- 暂放详细页候选，不抢前排

### 导购员统计

问题：

- 适合做员工复盘
- 但对老板当天补货、利润、库存判断帮助有限

建议：

- 后面如要强化店员绩效再接入

### 零售缴款单

问题：

- 对对账有用
- 但日常经营收益不如流水、销售、库存结构高

建议：

- 暂缓

### 储值按店汇总 / 储值卡汇总 / 储值卡明细

问题：

- 有经营价值，但前提是你们当前门店真的在高频使用储值体系
- 如果储值规模不大，这批表对老板首页的收益偏低

建议：

- 先不优先
- 等会员线成熟后再决定是否补
