# 小黑托昵店铺日常经营复盘表

版本：v1.0  
更新日期：2026-03-12  
适用对象：单城市门店、线下成交为主、抖音内容作为新增获客入口  
配套文档：
- `docs/06-店铺经营基础框架.md`
- `docs/04-项目推进路线图.md`

## 使用说明

- 这份文档的目标不是做复杂报表，而是让门店每天、每周、每月都有最小复盘动作。
- 默认先用最轻量的方式执行：
  - 每天关店前记一次
  - 每周固定复盘一次
  - 每月做一次总结
- 如果你后面想转成 Excel、Notion 或表格系统，也建议先按这份文档字段跑一段时间。

## 1. 每天复盘表

### 1.1 每天必须记录的 8 个数字

| 日期 | 到店人数 | 成交人数 | 抖音到店人数 | 问地址人数 | 问尺码/年龄人数 | 老客复购人数 | 今日最常被问问题 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 例如 2026-03-12 |  |  |  |  |  |  |  |

### 1.2 每天关店前 5 分钟要回答的 5 个问题

1. 今天抖音带来的人，进店后接住了吗
2. 今天哪类顾客最容易成交
3. 今天哪类问题被反复问到
4. 今天有没有顾客明确提到“刷到抖音来的”
5. 今天最值得明天继续拍的一条问题是什么

### 1.3 每天复盘结论模板

当天用这一句总结：

> 今天抖音带来的问题主要是 `[地址 / 尺码 / 最近适不适合来]`，门店成交更集中在 `[哪类顾客 / 哪类产品]`，明天最值得继续放大的问题是 `[具体问题]`。

## 2. 每周复盘表

### 2.1 每周只看这 7 个结果

| 周次 | 总到店人数 | 总成交人数 | 抖音到店总人数 | 老客复购总人数 | 最常被问的 3 个问题 | 最带咨询的内容题型 | 下周要放大的 2 个重点 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 例如 第 1 周 |  |  |  |  |  |  |  |

### 2.2 每周必须做的 4 个判断

1. 哪类内容最能把问题带进店
2. 哪类顾客进店后最容易成交
3. 哪类问题最适合继续做成视频
4. 哪个接待环节最容易流失顾客

### 2.3 每周复盘结论模板

> 本周内容最有效的是 `[题型]`，因为它带来了 `[地址 / 尺码 / 到店时机]` 类问题；门店最该加强的是 `[接待 / 尺码判断 / 到店承接 / 老客复购]`，下周重点继续放大 `[题型1]` 和 `[题型2]`。

## 3. 每月复盘表

### 3.1 每月重点看趋势，不只看单日

| 月份 | 抖音到店趋势 | 成交趋势 | 老客复购趋势 | 最稳定带客题型 | 最稳定成交产品逻辑 | 最需要优化的经营环节 |
| --- | --- | --- | --- | --- | --- | --- |
| 例如 2026-03 |  |  |  |  |  |  |

### 3.2 每月只回答 3 个核心问题

1. 抖音和门店有没有真正形成闭环
2. 哪类问题正在稳定带来成交
3. 经营上最需要补的是接待、产品结构、复购，还是私域承接

## 4. 每天 / 每周 / 每月的最小动作

### 每天

- 记 8 个数字
- 写 1 句总结
- 选 1 个明天继续拍的问题

### 每周

- 看 7 个结果
- 定下周放大的 2 个重点
- 把最常被问的 3 个问题同步给内容线

### 每月

- 看趋势
- 定门店经营的 1 个重点优化方向
- 定内容线下个月继续放大的 1 个方向

## 5. 最值得同步给内容线的经营信息

门店复盘后，优先把下面 4 类信息送回抖音内容：

1. 本周最常被问的 3 个问题
2. 本周最容易成交的顾客类型
3. 本周最容易让顾客纠结的尺码 / 厚薄点
4. 本周最值得拍成“到店证明”的真实片段

## 6. 这份复盘表怎么和内容仓联动

固定联动方式：

1. 门店每天复盘
2. 把高频问题给内容线
3. 内容线次日拍视频回应
4. 视频再把顾客带进店

## 7. 当前执行建议

如果你今天就开始用，先不要追求完整。  
先做到这 3 条：

1. 每天记 `抖音到店人数`
2. 每天记 `问地址 / 尺码 / 到店时机的人数`
3. 每天记 `最常被问的 1 个问题`

只要这 3 条连续记录 14 天，你的门店经营线就已经开始从“凭感觉”转向“有依据”。
