# 库存销售看板工具使用说明

版本：v1.2  
更新日期：2026-03-13  
适用对象：老板 / 老板娘、店员、数据协作人员

## 这是什么

这是一个基于 Python 的库存销售看板工具。  
它会读取门店导出的库存、销售、会员、导购、进销存报表，然后自动生成：

1. 一页 HTML 图形化看板
2. 一份 Markdown 文字摘要
3. 一份经营分析报告 Markdown
4. 一份补货建议清单 CSV
5. 一份去化建议清单 CSV
6. 一份品类风险概览 CSV

目标是让非专业人员也能快速看懂：

- 现在卖得怎么样
- 库存是不是太重
- 哪些要补
- 哪些要去化
- 哪些品类正在拖库存

默认口径说明：

- `道具` 已从主经营指标里单独剥离
- 看板里的 `经营销售额 / 经营库存额 / 库存覆盖天数 / 补货 / 去化` 默认都不包含道具
- 道具只在单独的参考区显示，方便看数值，但不干扰经营判断
- 看板里的 `北京时间与季节决策` 会按北京时间自动给出 `今天 / 本周 / 本月` 的经营动作
- 看板里的 `具体操作方案` 会按当前数据自动给出促销、去化、补货、会员经营等可执行方案

## 脚本位置

- `scripts/build_inventory_dashboard.py`

## 输入数据

最方便的方式是直接给一个 zip 压缩包，里面放系统导出的原始报表。  
脚本目前会自动识别这些类型：

1. 销售清单
2. 库存明细统计
3. 库存零售统计
4. 进销存统计
5. 导购员报表
6. 会员综合分析
7. 商品销售情况
8. 出入库单据

```bash

  --zip-file /Users/lizhihui/Desktop/库存综合分析报表【2026-03-13】.zip
```

## 这几个输出分别怎么看

### HTML 看板

最适合老板每天打开。  
重点看：

1. 经营销售额
2. 经营库存额
3. 经营健康灯
4. 北京时间与季节决策
5. 具体操作方案
6. 道具参考区
7. 会员销售额占比
8. 预计库存覆盖天数
9. 建议补货 SKU 数
10. 建议去化 SKU 数

### 经营健康灯怎么理解

- `红灯`：优先处理，说明已经明显影响经营判断或营业额
- `黄灯`：需要连续跟踪，说明风险已经出现
- `绿灯`：当前相对稳定，先维持节奏

### 北京时间与季节决策怎么看

- `今天先做`：适合今天营业前或营业中立刻执行
- `本周重点`：适合排班、补货、陈列、会员触达
- `本月方向`：适合老板做进货、去化、活动和结构调整

### 具体操作方案怎么看

- 如果看到 `库存周期过长` 这类方案，优先执行里面的 3 套促销 / 去化方案
- 如果看到 `负库存先纠偏`，先校库存，不要直接补货
- 如果看到 `畅销款缺货风险高`，先做补货分级，不要平均补
- 如果看到 `会员占比高`，优先做老客复购和定向触达

### Markdown 摘要

最适合发群里或者做日报。  
重点看：

1. 核心指标
2. 自动提醒
3. 优先补货 Top 5
4. 优先去化 Top 5

### 经营分析报告

最适合老板自己看或者周会上讲。  
重点看：

1. 一句话结论
2. 核心经营判断
3. 高压货风险品类
4. 导购表现靠前人员
5. 下一步建议

### 补货建议清单

优先看：

1. 销售金额高
2. 库存低
3. 周均销量高
4. 建议补货量

### 去化建议清单

优先看：

1. 实际库存高
2. 近期零售低
3. 动销率低
4. 建议动作

### 品类风险概览

优先看：

1. 库存金额 / 销售金额
2. 库存量 / 销售量
3. 状态是 `高压货` 还是 `需关注`

## 现阶段的判断规则

工具当前会先做这些判断：

1. `建议补货`
   适合销量不差、库存很低、库存周数太短的商品
2. `建议去化`
   适合库存很高、近期动销弱的商品
3. `高压货品类`
   适合库存额明显高于近期销售额的品类

这是一版实用规则，不是最终算法。  
后面可以继续按你们门店习惯细化。
