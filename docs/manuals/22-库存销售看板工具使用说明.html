库存销售看板工具使用说明¶

版本：v1.3
更新日期：2026-03-14
适用对象：老板 / 老板娘、店员、数据协作人员

这是什么¶

这是一个基于 Python 的库存销售看板工具。
它会读取门店导出的库存、销售、会员、导购、进销存报表，然后自动生成：

  1. 一页 HTML 图形化看板
  2. 一份 Markdown 文字摘要
  3. 一份经营分析报告 Markdown
  4. 一份补货建议清单 CSV
  5. 一份去化建议清单 CSV
  6. 一份品类风险概览 CSV

目标是让非专业人员也能快速看懂：

  • 现在卖得怎么样
  • 库存是不是太重
  • 哪些要补
  • 哪些要去化
  • 哪些品类正在拖库存

默认口径说明：

  • 道具 已从主经营指标里单独剥离
  • 看板里的 经营销售额 / 经营库存额 / 库存覆盖天数 / 补货 / 去化 默认都不包含道具
  • 道具只在单独的参考区显示，方便看数值，但不干扰经营判断
  • 店铺零售清单 里如果 输入人 不一致，默认视为不同店铺
  • 当前主逻辑固定关注 郭文攀 / 咸阳沣西吾悦专卖店
  • 其他输入人和店铺只做参考，不参与主经营结论
  • 看板里的 北京时间与季节决策 会按北京时间自动给出 今天 / 本周 / 本月 的经营动作
  • 看板里的 具体操作方案 会按当前数据自动给出促销、去化、补货、会员经营等可执行方案
  • 看板默认遵循 先看品类，再看 SKU，避免被某一个单款带偏

脚本位置¶

  • scripts/build_inventory_dashboard.py

输入数据¶

最方便的方式是直接给一个 zip 压缩包，里面放系统导出的原始报表。
脚本目前会自动识别这些类型：

  1. 销售清单
  2. 库存明细统计
  3. 库存零售统计
  4. 进销存统计
  5. 导购员报表
  6. 会员综合分析
  7. 商品销售情况
  8. 出入库单据

最常用的运行方式¶

python3 /Users/lizhihui/Workspace/Black\ Tony/scripts/build_inventory_dashboard.py \
  --zip-file /Users/lizhihui/Desktop/库存综合分析报表【2026-03-13】.zip

如果你已经提前解压到了目录，也可以直接跑目录：

python3 /Users/lizhihui/Workspace/Black\ Tony/scripts/build_inventory_dashboard.py \
  --input-dir /Users/lizhihui/Workspace/Black\ Tony/data/inventory_zip_extract

如果一个压缩包里有多家门店数据，也可以手动指定门店：

python3 /Users/lizhihui/Workspace/Black\ Tony/scripts/build_inventory_dashboard.py \
  --zip-file /Users/lizhihui/Desktop/库存综合分析报表【2026-03-13】.zip \
  --store 咸阳沣西吾悦专卖店

输出位置¶

默认输出到：

  • reports/inventory_dashboard/
  • docs/dashboard/

你会看到这些文件：

  1. 库存销售看板_日期.html
  2. index.html
  3. 库存销售摘要_日期.md
  4. 库存销售分析报告_日期.md
  5. 补货建议清单_日期.csv
  6. 去化建议清单_日期.csv
  7. 品类风险概览_日期.csv

说明：

  • 库存销售看板_日期.html：保留每天的历史版本
  • index.html：永远指向最新版本，适合后面发布到 GitHub Pages
  • reports/inventory_dashboard/：历史版本和本地留档
  • docs/dashboard/：GitHub Pages 用的固定发布目录

这几个输出分别怎么看¶

HTML 看板¶

最适合老板每天打开。
老板建议按这个顺序看：

  1. 老板一分钟结论
  2. 经营健康灯
  3. 补货重点品类 / 去化重点品类 / 跨季处理重点品类
  4. 补货 SKU 明细 / 去化 SKU 明细 / 跨季处理 SKU 明细
  5. 其他店铺参考

重点看：

  1. 经营销售额
  2. 经营库存额
  3. 老板一分钟结论
  4. 经营健康灯
  5. 北京时间与季节决策
  6. 具体操作方案
  7. 道具参考区
  8. 会员销售额占比
  9. 预计库存覆盖天数
  10. 建议补货 SKU 数
  11. 建议去化 SKU 数

如果后面发布到 GitHub Pages：

  • 优先发布 docs/dashboard/index.html
  • 这样老板访问的会是固定网址，不需要每天换链接
  • 推荐路径结构：
  • Pages 根目录用 docs/
  • 在线仪表盘固定访问 /dashboard/
  • 历史版本继续留在仓库的 reports/，不直接作为 Pages 首页
  • 真正开启前，先运行一次：
python3 /Users/lizhihui/Workspace/black\ Tony/scripts/check_pages_ready.py

如果你想按清单一步一步核对，再看：

老板一分钟结论怎么看¶

  • 结论：今天门店最该先做什么
  • 今天先做：今天营业中直接执行的动作
  • 今天先别做：容易做错的动作
  • 看板阅读顺序：按这个顺序往下看，不容易误判
  • 老板晨会可直接说：可以直接拿去和员工开口
  • 表格里的红黄绿标签支持 鼠标悬浮
  • 悬浮后会看到更详细的解释，例如 先停补再去化 会告诉你为什么这么做、下一步该怎么做

经营健康灯怎么理解¶

  • 红灯：优先处理，说明已经明显影响经营判断或营业额
  • 黄灯：需要连续跟踪，说明风险已经出现
  • 绿灯：当前相对稳定，先维持节奏

北京时间与季节决策怎么看¶

  • 今天先做：适合今天营业前或营业中立刻执行
  • 本周重点：适合排班、补货、陈列、会员触达
  • 本月方向：适合老板做进货、去化、活动和结构调整
  • 这部分是 季节节奏
  • 老板一分钟结论 是 今天操作顺序
  • 两者要一起看，但先看老板一分钟结论

具体操作方案怎么看¶

  • 如果看到 库存周期过长 这类方案，优先执行里面的 3 套促销 / 去化方案
  • 如果看到 负库存先纠偏，先校库存，不要直接补货
  • 如果看到 畅销款缺货风险高，先做补货分级，不要平均补
  • 如果看到 会员占比高，优先做老客复购和定向触达

Markdown 摘要¶

最适合发群里或者做日报。
重点看：

  1. 老板一分钟结论
  2. 核心指标
  3. 自动提醒
  4. 补货重点品类 Top 5
  5. 去化重点品类 Top 5

经营分析报告¶

最适合老板自己看或者周会上讲。
重点看：

  1. 老板一分钟结论
  2. 一句话结论
  3. 核心经营判断
  4. 高压货风险品类
  5. 导购表现靠前人员
  6. 下一步建议

补货建议清单¶

先记住一个原则：

  • 不要一上来就看 补货 SKU 明细
  • 先看 补货重点品类
  • 只有确定这个品类今天要补，才去看具体 SKU

优先看：

  1. 销售金额高
  2. 库存低
  3. 周均销量高
  4. 建议补货量

去化建议清单¶

先记住一个原则：

  • 不要一上来就看 去化 SKU 明细
  • 先看 去化重点品类
  • 只有确定这个品类今天要去化，才去看具体 SKU

优先看：

  1. 实际库存高
  2. 近期零售低
  3. 动销率低
  4. 建议动作

品类风险概览¶

优先看：

  1. 库存金额 / 销售金额
  2. 库存量 / 销售量
  3. 状态是 高压货 还是 需关注

现阶段的判断规则¶

工具当前会先做这些判断：

  1. 建议补货 适合销量不差、库存很低、库存周数太短的商品
  2. 建议去化 适合库存很高、近期动销弱的商品
  3. 高压货品类 适合库存额明显高于近期销售额的品类

这是一版实用规则，不是最终算法。
后面可以继续按你们门店习惯细化。

现阶段最适合继续升级的方向¶

  1. 加入红黄绿预警
  2. 加入自动补货量模型
  3. 加入按尺码断码提醒
  4. 加入多门店对比
  5. 做成更简单的桌面点击版或网页小工具

当前结论¶

这套数据完全值得继续做成一个长期使用的 Python 工具。
因为它已经不只是“能看图”，而是已经能支撑：

  1. 补货
  2. 去化
  3. 会员经营
  4. 品类结构调整
  5. 老板每日看板

给 Claude review 时最值得看的点¶

  1. 看板是否真正按 老板可执行 的顺序组织，而不是只堆指标
  2. 是否明确区分了 主店铺逻辑 和 其他店铺参考
  3. 是否始终遵循 先看品类，再看 SKU
  4. 是否把 道具 从主经营口径里稳定剥离

如果你想看到 2025-03 到现在的完整趋势¶

当前这版看板已经同时支持：

  1. 最近一段销售明细趋势
  2. 历史累计销售额和累计销量

但如果要看到从 2025-03 到现在的完整逐日或逐月趋势，还需要补导更长时间范围的 销售清单。
